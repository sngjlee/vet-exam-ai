# Content Policy

> KVLE 문제은행 구축 과정에서 기출 기반 텍스트, 이미지, AI 재작성 결과물을 다루는 내부 운영 기준.
> 법률 자문을 대체하지 않으며, 공개/상업 서비스 전 리스크를 줄이기 위한 실무 원칙이다.

---

## 기본 원칙

- 수의사 국가시험 원문 문제와 vet40 등 기출 아카이브 이미지는 서비스 화면에 그대로 노출하지 않는다.
- AI 재작성 결과물도 자동 승인하지 않고, 정답 유지 여부와 수의학적 정확성을 검수한 뒤 사용한다.
- 원문 텍스트와 원본 이미지는 서비스 DB에 불필요하게 보관하지 않는다.
- 모든 문항은 출처, 재작성 모델, 프롬프트 버전, 검수 상태, import batch를 추적할 수 있어야 한다.
- 저작권 리스크와 의학적 정확도 리스크가 충돌하면 정확도를 우선하되, 해당 문항은 공개하지 않고 재설계한다.

---

## 텍스트 문항

- 원문을 단순 치환하거나 문장만 바꾼 수준의 문항은 승인하지 않는다.
- 문제의 평가 목표, 정답, 핵심 개념은 유지하되 표현, 상황, 조건, 해설은 독립적으로 재구성한다.
- 정답 근거가 애매해지거나 복수 정답 가능성이 생긴 문항은 `rejected` 또는 `needs_review` 상태로 둔다.
- 선택지 중복, 선택지 길이 편향, 정답 위치 패턴, 해설 누락은 자동 검증 대상에 포함한다.

---

## 최신 회차

- 67~70회처럼 기존 아카이브에 빠진 최신 회차는 출제 경향 파악과 커버리지 확보를 위해 intake 대상에 포함한다.
- 최신 회차라도 원문 텍스트와 원본 이미지를 서비스 DB에 그대로 저장하거나 사용자에게 노출하지 않는다.
- 회차별 진행 상태는 `not_started`, `intake_done`, `rewrite_done`, `reviewing`, `approved`, `blocked`로 관리한다.
- 최신 회차는 사용자 가치가 높지만 저작권과 정확도 검수 기준은 완화하지 않는다.
- 매년 새 국가시험이 시행되면 같은 절차로 intake, 재작성, 검수, 반영 여부를 판단한다.

---

## 이미지 문항

- 원본 이미지를 입력으로 한 단순 스타일 변환, 노이즈 추가, 색감 변경, 크롭 변경은 승인하지 않는다.
- 이미지 포함 문항은 먼저 아래 유형 중 하나로 분류한다.
  - `diagram_rebuild`: 새 도식으로 재작성 가능
  - `licensed_asset_needed`: 라이선스 확보 또는 공개 라이선스 자료 필요
  - `question_rewrite_needed`: 이미지 없이 풀 수 있게 문항 재설계 필요
  - `remove_image`: 이미지 의존도를 제거하거나 문항 제외
- 해부도, 생리도, 표, 알고리즘은 원본 구도에 의존하지 않는 새 도식으로 만든다.
- 병변 사진, 방사선, 초음파, 조직 슬라이드는 AI 재생성보다 라이선스 자료, 자체 제작 자료, 공개 라이선스 자료를 우선한다.
- 이미지가 진단 단서인 경우 새 이미지 사용 후 정답이 바뀌지 않는지 별도 검수한다.

---

## 공개 라이선스 자료

- 공개 라이선스 이미지를 사용할 때는 원본 URL, 라이선스 종류, 접근일, 저작자 표시 필요 여부를 기록한다.
- 라이선스 조건이 불명확하거나 상업적 이용 가능 여부가 애매하면 사용하지 않는다.
- 출처 표기가 필요한 자료는 서비스 화면 또는 내부 출처 화면에서 표시 가능한 구조를 준비한다.

---

## 검수 상태

- `raw_imported`: 원천 데이터가 들어온 상태
- `ai_rewritten`: AI 재작성 완료
- `auto_validated`: 구조 검증 통과
- `human_reviewed`: 사람이 정답과 해설을 검수
- `image_pending`: 이미지 권리 또는 정확도 검수 대기
- `approved`: 사용자에게 노출 가능
- `rejected`: 폐기 또는 전면 재작성 필요

사용자에게 노출되는 문항은 원칙적으로 `approved` 상태만 허용한다.

---

## 업로드 / 롤백

- 모든 대량 업로드는 `import_batch_id`를 가진다.
- 업로드 전 dry-run으로 신규, 수정, 충돌, 비활성화 예정 문항 수를 확인한다.
- 문제가 있는 batch는 문항 단위 수정 전에 batch 단위 비활성화가 가능해야 한다.
- 대량 업로드 후에는 과목별 샘플을 실제 앱 플로우에서 확인한 뒤 다음 batch로 진행한다.

---

## 보류 기준

아래에 해당하면 공개하지 않는다.

- 원문과 실질적으로 유사한 표현이 많이 남아 있는 문항
- 원본 이미지의 구도나 특징을 사실상 유지한 이미지
- 정답이 불명확하거나 복수 정답 가능성이 있는 문항
- 해설이 정답을 충분히 뒷받침하지 못하는 문항
- 출처, 재작성 이력, 검수 상태가 누락된 문항
- 공개 라이선스 또는 사용 권한이 불명확한 이미지 자료
