# Seeding Plan (draft v0.1)

> KVLE 베타 런칭(2026-07-01) 시점의 초기 콘텐츠/사용자/커뮤니티 시드 전략 초안.
> 정식 공개(2026-08-31)와 시험일(2027-01-15)을 향한 운영 시퀀스 기준이다.
> 마케팅 전략이 아니라 "비어 있는 화면을 어떻게 채울 것인가"에 대한 운영 계획이다.

---

## 전제

- 베타 시점의 핵심 자산: 검수된 문제 ~2,000건(rewrite 200/200 완료 시 ~2,500~3,000건 예상), 20개 과목 커버.
- 커뮤니티(댓글, 암기법, 정정 제안)는 베타 시작 시점에 0건이다.
- 첫 사용자는 빈 댓글 칸을 본다. 빈 칸이 곧 "이 서비스는 죽었다"는 신호로 읽히지 않게 만드는 것이 시드의 1차 목적이다.

## 콘텐츠 시드

### 검수 문제 (자동 시드)

- `pipeline/upload.py --all`로 rewrite 결과를 원격 적재 → questions 테이블 채움
- 베타 시작 전: 200/200 rewrite 완료 + active 카운트 ≥2,500 목표
- 이미지 의존 문항(현재 ~210건)은 active=false 유지, 2차 페이즈에서 이미지 처리 후 활성화

### 시드 댓글/암기법

목적: 첫 사용자가 어떤 문항을 열어도 최소 1개의 사례 댓글을 보도록 한다.

- 운영팀이 시드 사용자에게 의뢰: 본인 학습 노트 중 공유 가능한 암기법을 댓글로 등록
- 1인당 20~30건 × 시드 사용자 10~15명 = 200~400건 목표
- 분포 원칙:
  - 과목 균형 (한 과목에 몰리지 않게)
  - 회차 균형 (최신 회차 우선, 그러나 한 회차에 몰리지 않게)
  - 댓글 유형 균형 (암기법:정정:풀이토론 = 6:2:2 정도)
- 시드 댓글에는 작성자 표시를 그대로 둔다 ("KVLE 운영팀"으로 가짜 표기하지 않음 — 신뢰 훼손).
- 시드 댓글도 일반 사용자와 동일한 가이드라인을 적용받는다.

### 시드 정정 제안

- 운영팀이 rewrite 검수 과정에서 발견한 잠재 오류를 정정 제안 형식으로 미리 등록
- "정정 제안 → 채택 → 문항 반영" 워크플로우가 살아 있음을 보여줌
- 베타 시작 시점에 채택 5~10건 + pending 5~10건 표시

## 사용자 시드

### Phase 1: 시드 사용자 (D-30 ~ D-15, 2026-06-01 ~ 06-15)

대상:
- 운영팀 본인의 1차 네트워크(같은 학년/같은 학교 수험 동료)
- 수의대생 커뮤니티 운영진 중 협조 가능한 인원
- 본인 학습 노트가 정리되어 있어 시드 콘텐츠 기여가 가능한 사람

규모: 10~20명

기여:
- 시드 댓글/암기법 등록
- UI/UX 사용성 피드백
- 첫 정정 제안 등록

대가:
- 베타 평생 무료 등급(향후 유료화 시)
- 정정 채택 누적에 따른 배지

### Phase 2: 베타 초청 사용자 (D-15 ~ D-Day, 2026-06-15 ~ 07-01)

대상:
- 시드 사용자의 1차 네트워크 추천
- 수의대 4학년 + 졸업생 시험 준비자 우선

규모: 100~200명

배포 채널:
- 시드 사용자별 초청 코드 5~10개 발급
- 수의대 단톡방, 학년 카페 등 시드 사용자가 직접 공유
- 외부 광고/홍보는 본 단계에서 진행하지 않음

### Phase 3: 일반 공개 (2026-08-31 ~)

- 초청 코드 폐지 (또는 선택 항목)
- 일반 가입 허용 (privacy-signup-policy.md 기본선 적용)

## 커뮤니티 활성화 운영

### 첫 4주 (베타)

운영팀이 매일 수행:
- 신규 사용자 가입 알림 확인 → 환영 댓글(템플릿 아닌 개별 응답) 1회
- 신규 댓글 24시간 이내 reviewer 1명이 읽고 1건 이상 좋아요 또는 답글
- 정정 제안은 7일 이내 처리 (community-guideline에서 약속한 SLA)
- 가이드라인 위반 0건 유지, 위반 시 즉시 처리 + 사례 내부 공유

### 시드 콘텐츠와 자생 콘텐츠 비율 추적

- 주간 단위로 (운영팀 작성 / 일반 사용자 작성) 비율 측정
- 베타 4주차에 자생 비율 ≥30% 목표
- 베타 8주차(정식 공개 시점)에 자생 비율 ≥60% 목표
- 비율이 안 오르면 시드 사용자 추가 유치 vs 시드 콘텐츠 추가 등록 중 선택

## 핵심 지표

| 지표 | 베타 시작 (D-Day) | 정식 공개 (D+60) | 시험 D-45 |
|---|---|---|---|
| 활성 문항 | ≥2,500 | ≥2,800 | ≥3,000 |
| 댓글 누적 | 200~400 (시드) | 1,500+ | 5,000+ |
| 정정 채택 누적 | 5~10 (시드) | 30+ | 100+ |
| 주간 활성 사용자 (WAU) | 50+ | 300+ | 1,000+ |
| 댓글 자생 비율 | 0% | ≥60% | ≥90% |
| 신고 처리 SLA 준수율 | — | ≥95% | ≥95% |

수치는 초안 가정이며, 베타 1주차 데이터로 재보정한다.

## 위험과 대응

| 위험 | 대응 |
|---|---|
| 시드 사용자가 댓글을 안 씀 | 1:1 요청 + 작성 가이드 템플릿 제공 + 운영팀이 직접 작성 후 시드 사용자가 보강 |
| 첫 일반 사용자가 빈 화면을 봄 | "검수된 문제 + 운영팀 시드 댓글" 우선 노출, "신규 댓글" 탭은 자생 콘텐츠 누적 후 활성화 |
| 시험 임박 시점에 신고/정정 폭주 | reviewer 인원 사전 확보, 자동 비공개 임계치 운영 (moderation-playbook 참조) |
| 저작권 분쟁 신고 접수 | 즉시 비공개 + 법무 자문 절차 가동, 외부 카피 가드 사전 점검 |
| 시드 사용자가 가이드라인 위반 | 일반 사용자와 동일하게 처리. 시드 신분 면책 없음. 운영팀 신뢰 일관성이 더 중요 |

## 시드 종료 시점

다음 조건 모두 충족 시 시드 운영 종료를 검토한다.

- 댓글 자생 비율 ≥80%
- WAU ≥500
- 정정 제안 자생 ≥주 10건
- reviewer 큐가 SLA 내 자동 소화

조건 미충족 시 시드 단계를 연장한다. 종료 시점에 시드 사용자에게 별도 안내와 감사 표시를 한다.

- 학교/학년 단위 단체 가입 처리 방식
- 정식 공개 후 초청 코드 유지 여부
- 시험 후(2027-01-16 이후) 운영 모드 전환 (다음 회차까지의 idle 기간 운영)
