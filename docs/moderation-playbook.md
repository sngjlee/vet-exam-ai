# Moderation Playbook (draft v0.1)

> KVLE 커뮤니티 모더레이션 운영 매뉴얼 초안.
> 신고 처리, 자동 비공개, 정정 제안 검수, admin 액션 로그 활용에 대한 내부 워크플로우 기준이다.
> community-guideline.md를 사용자 규범, 본 문서를 운영자 규범으로 사용한다.

---

## 역할

- `admin`: 모든 모더레이션 액션 권한, 사용자 정지 결정권
- `reviewer`: 정정 제안 채택/반려, 신고 1차 분류, 게시물 비공개 권한
- `learner`: 일반 사용자

reviewer는 admin 승격 전 검증 단계로 운영한다. 신규 reviewer는 첫 30일간 admin이 처리 결과를 샘플링한다.

---

## 신고 처리 SLA

- P0 (즉시 차단): 신상 정보 노출, 시험 부정행위 권유, 명백한 저작권 침해 — 4시간 이내 1차 조치
- P1 (당일 처리): 인신공격, 광고, 스팸, 명백한 가이드라인 위반 — 24시간 이내
- P2 (주간 처리): 표현 톤 문제, 근거 부족, 경계선 사례 — 7일 이내

SLA 초과는 admin_audit_logs에 기록되고, 주간 리뷰에서 원인을 검토한다.

---

## 신고 처리 워크플로우

1. **접수 확인** — 신고 큐(`comment_reports.status = 'pending'`)에서 가장 오래된 P0/P1 우선
2. **컨텍스트 수집** — 대상 글, 작성자 이력(이전 위반 0/1/2회 이상), 신고자 이력(허위 신고 패턴)
3. **분류 결정** — 아래 표 기반
4. **액션 실행** — 비공개, 작성자 안내, 신고자 안내, audit log 기록
5. **마감 처리** — `comment_reports.status = 'resolved'`, 처리 사유 코드 기록

| 상황 | 1차 액션 | 2차 액션 |
|---|---|---|
| 명백한 가이드라인 위반 | 글 비공개 + 작성자 안내 | 반복 시 계정 제한 검토 |
| 경계선 사례 | reviewer 1명 더 검토 요청 | 합의 시 처리, 미합의 시 admin 결정 |
| 신고 사유 부적절 | 신고 반려 + 신고자 안내 | 반복 시 신고 기능 제한 |
| 작성자/신고자 동일인 의심 | admin에게 에스컬레이션 | 확인 후 양쪽 처리 |
| 법적 위험 (저작권, 명예훼손) | 즉시 비공개 + admin 호출 | 법무 자문 요청 여부 결정 |

## 자동 비공개 트리거

`comment_votes` 음수 누적, `comment_reports` 신고 누적이 임계치를 넘으면 자동 비공개 후 reviewer 큐로 전달한다.

- 신고 누적 임계치: 같은 글에 서로 다른 사용자 5명 이상 신고 → 자동 비공개
- 음수 vote 임계치: 게시 24시간 이내 -10 이하 → 자동 비공개
- 자동 비공개는 작성자에게 "검토 중" 상태로 표시되며, reviewer가 24시간 이내 결정한다.
- 자동 비공개 후 reviewer가 "정상"으로 판정하면 즉시 복원한다. 복원은 audit log에 사유 코드와 함께 기록한다.
- vote DELETE 시에도 누적 카운트가 재계산되도록 트리거가 동작해야 한다(community-tables migration에서 처리됨).

임계치는 베타 운영 첫 4주간 데이터를 보고 조정한다.

## 정정 제안 검수

`question_corrections.status = 'pending'` 큐를 reviewer가 처리한다.

처리 옵션:
- `accepted`: 문항 본문/정답/해설에 반영 + 작성자 표시 + audit log
- `rejected`: 사유 코드와 함께 반려 + 작성자 안내
- `needs_more_info`: 작성자에게 근거 보강 요청 (1회만)

판정 기준:
- 정답이 변경되는 정정은 reviewer 2명 이상 합의 또는 admin 승인 필요
- 본문 오타, 표현 명료화는 reviewer 1명 판단으로 채택 가능
- 출처가 명시되지 않은 정정은 기본 `needs_more_info` 처리, 7일 무응답 시 `rejected`
- 같은 문항에 같은 정정 제안이 2회 이상 들어오면 우선순위 상향

채택된 정정은 문항 버전을 올리고, 정정 전후 본문을 audit log에 보존한다.

## 사용자 제재

| 단계 | 조건 | 조치 | 해제 |
|---|---|---|---|
| 경고 | 1차 위반 | 안내 메시지 + 글 비공개 | 자동 |
| 댓글 제한 7일 | 동일 사유 2차 위반 또는 P1 위반 | 댓글 작성 차단, 읽기는 가능 | 7일 후 자동 |
| 댓글 제한 30일 | 동일 사유 3차 위반 또는 P0 위반 | 댓글 작성 차단 | 30일 후 admin 검토 후 해제 |
| 계정 일시 정지 | 4차 위반 또는 심각한 단일 위반 | 로그인 후 안내 페이지만 노출 | admin 검토 후 해제 |
| 계정 영구 정지 | 5차 위반, 시험 부정행위 권유, 반복적 신상 노출 | 로그인 차단 | 해제 없음. 이의 신청 1회 가능 |

모든 제재는 admin_audit_logs에 다음을 기록한다.
- 대상 사용자 ID
- 단계
- 적용 사유 코드
- 근거 게시물 ID
- 적용/해제 시각
- 처리 admin/reviewer ID

## 에스컬레이션

reviewer가 admin에게 에스컬레이션해야 하는 경우:
- 정답이 바뀌는 정정 제안 채택 결정
- 계정 정지 결정 (일시/영구 무관)
- 법적 위험이 있는 게시물 (명예훼손, 저작권, 약관 외 표현)
- 같은 사용자에 대한 신고가 단기간(7일) 내 5건 이상
- 처리 방향이 reviewer 간 1:1로 갈리는 경우

admin이 외부 자문(법무, 수의학 전문)을 요청해야 하는 경우:
- 임상 정확도와 시험 정답이 충돌하는 정정
- 외부 콘텐츠 인용의 합법성 판단
- 사용자 신원 노출 사고

---

## 일일/주간 운영

- **일일 (reviewer 당직)**: P0/P1 신고 큐 0건 유지, 정정 제안 큐 7일 이상 항목 0건 유지
- **주간 (admin)**: SLA 초과 건수 리뷰, 자동 비공개 임계치 조정 검토, 반복 위반자 제재 단계 검토
- **월간 (admin)**: 가이드라인 변경 필요성 검토, reviewer 처리 일관성 샘플링, 신고-반려 비율 추세 확인

---

## 모더레이터 행동 강령

- 자신이 작성한 글에 대한 신고/정정 제안은 직접 처리하지 않는다.
- 친분이 있는 사용자에 대한 제재 결정은 다른 reviewer/admin에게 위임한다.
- 사용자 개인정보(이메일, IP, 로그인 기록)는 모더레이션 목적 외로 조회하지 않는다.
- 모든 액션의 사유를 코드로 기록한다. "그냥"은 사유가 아니다.
- 본인의 판단이 흔들리면 에스컬레이션한다. 단독 판단을 강요받지 않는다.

---

## 투명성

- 분기별로 신고 처리 통계(접수/처리/평균 처리 시간/제재 건수)를 공개한다.
- 가이드라인 변경 이력을 공개 페이지에 보존한다.
- 개별 처리 사유는 작성자에게만 안내하고 외부 공개하지 않는다.

---

## 미정 항목

- 자동 비공개 임계치 구체 수치 (베타 4주 데이터 후 결정)
- reviewer 보상 체계 (현재 자원봉사 가정)
- 외부 신고(저작권자, 정부기관) 처리 절차
- 데이터 보존 기간(신고 기록, audit log)
