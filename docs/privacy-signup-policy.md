# Privacy and Signup Policy

> KVLE 회원가입, 개인정보 수집, 가입 대상, 자격 인증에 대한 내부 운영 기준.
> 법률 자문을 대체하지 않으며, 공개 서비스 전 개인정보 리스크를 줄이기 위한 실무 원칙이다.

---

## 기본 판단

- 초기 서비스는 이메일과 비밀번호 기반 가입으로 충분하다.
- 이름, 휴대전화번호, 학교, 학번, 학생증, 재학증명서, 면허번호는 초기 필수 수집 항목으로 두지 않는다.
- 개인정보는 "나중에 쓸 수도 있음"이 아니라 "지금 기능 제공에 반드시 필요함"이 설명될 때만 수집한다.
- 수의대생 전용 폐쇄 커뮤니티가 아니라 시험 대비 학습 도구라면, 초기에는 일반 가입을 허용한다.
- 장기적으로 동물보건사 시험까지 확장할 수 있도록 사용자 자격보다 학습 트랙을 중심으로 설계한다.

---

## 초기 가입 필수 항목

- 이메일
- 비밀번호
- 이용약관 동의
- 개인정보 처리방침 동의

서비스 이용 과정에서 자동 생성될 수 있는 정보:

- 사용자 ID
- 로그인 기록
- 문제 풀이 기록
- 오답노트, SRS 복습 상태
- 접속 로그, 기기/브라우저 정보, IP 주소

자동 생성 정보도 개인정보 또는 개인정보에 준하는 운영 데이터로 보고, 처리 목적과 보유기간을 문서화한다.

---

## 선택 수집 항목

아래 항목은 기능이 생기기 전까지 수집하지 않는다.

- 닉네임: 랭킹, 커뮤니티, 공개 프로필이 생길 때 선택 수집
- 시험 예정일: D-day와 학습 플랜 기능을 사용할 때 선택 수집
- 준비 시험: 수의사 국가시험, 동물보건사 시험 등 학습 트랙 선택 시 수집
- 학교/소속: 학교별 기능이나 인증 배지가 필요할 때 선택 수집
- 휴대전화번호: SMS 인증, 결제, 강한 본인확인이 필요할 때만 수집

선택 항목을 입력하지 않아도 핵심 학습 기능은 사용할 수 있어야 한다.

---

## 동의 UX

- 이용약관 동의와 개인정보 처리방침 동의는 필수 동의로 분리해 표시한다.
- 마케팅 수신 동의는 선택 동의로 분리하고 기본값을 미동의로 둔다.
- 맞춤형 분석, 연구/통계 활용, 광고성 알림은 각각 필요한 경우에만 별도 선택 동의를 받는다.
- 필수 동의와 선택 동의를 한 번에 포괄 동의시키지 않는다.
- 가입 화면에서 처리 목적, 수집 항목, 보유기간을 짧게 요약하고 전체 처리방침으로 연결한다.

## 가입 대상

초기 정책:

- 일반 가입 허용
- 서비스 설명은 "수의사 국가시험 대비 학습"으로 명확히 표시
- 수의대생 전용 자료실, 학교별 비공개 커뮤니티, 인증 배지는 초기 범위에서 제외

이유:

- 수의대생 여부를 확인하려면 학교 이메일, 학생증, 재학증명서, 학번 등 추가 개인정보가 필요하다.
- 인증 자료는 유출 시 피해가 크고, 수동 검수 운영 부담도 크다.
- 동물보건사 시험으로 확장하려면 처음부터 폐쇄형 수의대생 서비스로 고정하지 않는 편이 유리하다.

## 자격 인증

자격 인증은 아래 조건 중 하나가 충족될 때만 도입한다.

- 저작권 또는 라이선스 조건상 접근 제한이 필요함
- 학교/학년별 기능처럼 인증된 사용자에게만 제공해야 하는 기능이 생김
- 유료 B2B 또는 학교 제휴 모델이 생김
- 커뮤니티 악용 방지를 위해 인증 배지가 필요함

도입 순서:

1. 학교 이메일 인증
2. 일회성 관리자 검수와 인증 상태만 저장
3. 외부 본인확인 또는 증빙 업로드

학생증, 재학증명서, 학번은 최후의 수단이다. 증빙 파일을 받는 경우 원본 파일은 검수 후 즉시 삭제하고, 인증 여부와 검수 일시만 남기는 방향을 우선 검토한다.

## 학습 트랙 설계

장기 확장을 위해 가입 자격과 학습 트랙을 분리한다.

- `user_role`: `learner`, `admin`, `reviewer`
- `exam_track`: `veterinarian`, `veterinary_technician`
- `verification_status`: `none`, `student_verified`, `professional_verified`, `manual_verified`

수의사 국가시험과 동물보건사 시험은 문제은행, 과목, 난이도, 학습 플랜이 다르므로 `exam_track` 기준으로 분리한다.

## 미성년자

- 초기 서비스는 만 14세 미만 가입을 지원하지 않는 방향을 기본값으로 한다.
- 만 14세 미만 가입을 허용하려면 법정대리인 동의 절차와 아동용 개인정보 안내가 필요하다.
- 동물보건사 시험 준비생 중 미성년자가 있을 수 있으므로, 확장 전 연령 정책을 다시 검토한다.
- 생년월일을 기본 수집하지 않는 경우, 가입 화면에서 만 14세 이상 여부 확인 고지를 우선 적용한다.

## 탈퇴 / 파기

- 사용자가 탈퇴하면 이메일 등 계정 식별정보는 삭제한다.
- 문제 풀이 기록, 오답노트, SRS 상태는 사용자 복구 필요성이 없으면 삭제하거나 익명화한다.
- 신고, 결제, 분쟁 대응 등 법적 보관 필요성이 있는 정보는 목적과 기간을 별도로 명시한다.
- 장기 미접속 계정의 처리 기준은 운영 규모가 커지기 전 별도 정책으로 정한다.

---

## 보안 기본선

- 비밀번호는 Supabase Auth 등 인증 제공자의 안전한 해시 저장을 사용하고 직접 평문 저장하지 않는다.
- 관리자, 검수자, 일반 사용자의 권한을 분리한다.
- 관리자 화면 접근과 문항 수정 이력은 로그로 남긴다.
- 운영 DB와 개발/실험 DB를 분리한다.
- 개인정보 유출, 오발송, 권한 설정 오류가 발생했을 때의 대응 절차를 문서화한다.
