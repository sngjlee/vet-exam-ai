## Phase 0 — 문제은행 신뢰도 확보

> 수험생이 "이걸 풀면 실제로 도움이 된다"고 믿게 만드는 기반.  
> 현재 진행 중인 기출 파싱 → AI 재작성(Sonnet 4.6) → 검수 → Supabase 삽입 작업.

### 1. 콘텐츠 출처 / 처리 이력 기록
- 각 문항의 원천, 재작성 모델, 프롬프트 버전, 처리 일시, 검수 상태를 추적 가능하게 저장
- `source_type`, `source_ref`, `rewrite_model`, `rewrite_prompt_version`, `review_status`, `reviewed_at` 같은 메타데이터 설계
- 공개 서비스 전 문제가 생겼을 때 문항 단위로 원인 추적, 비활성화, 재검수할 수 있어야 함
- 원문 텍스트나 원본 이미지를 서비스 DB에 불필요하게 보관하지 않는 원칙 확정

### 2. 최신 회차 커버리지 정책
- 현재 아카이브가 66회까지라면 67~70회는 최신 출제 경향 파악을 위해 intake 대상에 포함
- 단, 최신 회차 원문/이미지를 서비스 DB에 그대로 저장하거나 노출하지 않고 처리 이력과 재작성 후보만 관리
- 회차별 수집 상태를 `not_started`, `intake_done`, `rewrite_done`, `reviewing`, `approved`, `blocked`로 추적
- 70회처럼 가장 최근 회차는 사용자 가치가 높지만 권리 리스크도 동일하므로 우선순위만 높이고 검수 기준은 완화하지 않음
- 최신 회차가 들어올 때마다 연 1회 업데이트 루틴으로 반복 가능해야 함

### 3. AI 재작성 결과물 검수
- 과목별 샘플 5~10문제씩 원문 대비 확인
- 정답 유지 여부, 선택지 중복/모순, 해설 정확도, 난이도 변형 여부 점검
- 이미지 포함 문제는 1차에서 `is_active=false`로 분리

### 4. 이미지 포함 문항 권리 리스크 처리
- vet40 등 기출 아카이브 원본 이미지는 그대로 사용하지 않고 별도 처리 대상으로 분리
- 이미지 유형을 `diagram_rebuild`, `licensed_asset_needed`, `question_rewrite_needed`, `remove_image`로 태깅
- 단순 해부도/생리도/표/알고리즘은 원본 구도에 의존하지 않는 새 도식으로 재작성
- 병변 사진, 방사선, 초음파, 조직 슬라이드는 AI 변형보다 라이선스 확보, 공개 라이선스 자료, 자체 제작 자료, 또는 이미지 비의존 문항으로 재설계
- 원본 이미지를 입력으로 한 단순 스타일 변환은 실질적 유사성 리스크가 남으므로 지양
- 이미지 생성/재작성 비용은 1,000장 기준 대략 수만~수십만 원 수준으로 보고, 실제 병목은 저작권 리스크 분류와 수의학적 정확도 검수로 관리

### 5. 검수 상태 워크플로우
- 단순 `is_active`만으로 관리하지 않고 `raw_imported`, `ai_rewritten`, `auto_validated`, `human_reviewed`, `image_pending`, `approved`, `rejected` 같은 상태로 분리
- 사용자에게 노출 가능한 상태와 내부 보류 상태를 명확히 구분
- 이미지 보류, 정답 의심, 해설 의심, 과목 분류 의심을 별도 플래그로 관리
- 관리자 또는 검수자가 상태를 바꾼 이력을 남겨 대량 수정 사고를 줄임

### 6. 자동 검증 스크립트
- `choices.length === 5` 확인
- `answer`가 choices 중 하나와 정확히 일치하는지 확인
- 문제/선택지/정답/해설 빈 값 검사
- `id`, `subject`, `category`, `session`, `round`, `year` 누락 및 중복 검사
- `review_status`, `source_type`, `rewrite_model`, `import_batch_id`, `exam_round` 같은 필수 운영 메타데이터 검사
- 전체 JSON 검증 후 통과한 문제만 업로드 후보로 분류

### 7. Supabase 스키마 확장
- `questions` 테이블에 `session`, `round`, `year`, `community_notes`, `tags`, `is_active` 컬럼 추가
- `review_status`, `source_type`, `source_ref`, `rewrite_model`, `rewrite_prompt_version`, `import_batch_id`, `asset_status`, `exam_round` 컬럼 추가 검토
- `source="past_exam"` 기준으로 기출 기반 문제 구분
- `id` 기준 upsert 가능하도록 중복 정책 확정

### 8. 대량 업로드 / 롤백 전략
- 모든 import를 `import_batch_id` 단위로 기록
- 업로드 전 dry-run으로 신규/수정/충돌/비활성화 예정 문항 수 출력
- 잘못 올라간 배치를 한 번에 비활성화하거나 이전 상태로 되돌리는 롤백 스크립트 준비
- 대량 업로드는 과목별 또는 회차별로 쪼개서 진행하고, 업로드 후 샘플 검수 통과 시 다음 배치 진행

### 9. 20과목 데이터 구조 반영
- 기존 5과목 중심 category를 수국시 20과목으로 확장
- 과목 선택, 통계, 약점 분석, 색상 팔레트가 20과목을 처리하도록 수정
- 교시별 필터(1교시/2교시/3교시/법규) 확장 고려

### 10. 소량 업로드 후 앱 플로우 테스트
- 예: `해부학` + `수의법규`처럼 성격이 다른 2개 과목 먼저 업로드
- 퀴즈, 오답노트, SRS 복습, 통계, 약점 집중 연습 정상 동작 확인
- 문제가 앱에서 학습 가능한 품질인지 실제 사용자 흐름으로 검증

## Phase 0.5 — 출시 전 개인정보 / 가입 정책

> 공개 서비스로 전환하기 전에 반드시 확정해야 하는 운영 기준.

### A. 개인정보 최소수집 회원가입
- 초기 가입 필수 항목은 이메일, 비밀번호, 약관 동의, 개인정보 처리방침 동의로 제한
- 이름, 휴대전화번호, 학교, 학번, 재학증명서, 수의대생 인증 자료는 기본 가입 단계에서 수집하지 않음
- 마케팅 수신, 맞춤형 분석, 연구/통계 활용은 필수 약관과 분리된 선택 동의로 관리
- 개인정보 처리 목적, 처리 항목, 보유기간, 파기 기준, 위탁/Supabase 등 외부 서비스 사용 여부를 가입 화면과 처리방침에 명시
- 탈퇴 시 계정 식별정보와 학습 기록의 삭제/익명화 기준을 사전에 정함

### B. 가입 대상 / 자격 인증 정책
- 초기 버전은 일반 가입을 허용하되, 서비스 포지셔닝은 "수의사 국가시험 대비 학습"으로 명확히 유지
- 수의대생 전용 기능이 필요해질 때만 별도 인증 배지를 도입하고, 전체 가입을 막는 방식은 보류
- 수의대생 인증은 학교 이메일 또는 일회성 검수처럼 개인정보가 적게 남는 방식부터 검토
- 재학증명서, 학생증, 학번 등 민감한 증빙 자료 업로드는 운영 부담과 유출 리스크가 커서 최후의 수단으로 둠
- 장기적으로 동물보건사 시험까지 확장할 수 있도록 `user_role`, `exam_track`을 분리해 설계

### C. 미성년자 / 아동 개인정보
- 만 14세 미만 가입은 초기에는 지원하지 않는 방향을 기본값으로 검토
- 만 14세 미만 가입을 허용하려면 법정대리인 동의 절차와 아동용 개인정보 안내가 필요하므로 별도 출시 범위로 분리
- 동물보건사 시험 준비생 중 미성년자 가능성을 고려해 생년월일 수집 없이도 연령 제한 고지가 가능한 UX를 우선 검토

### D. 보안 / 접근통제 기본선
- 비밀번호는 Supabase Auth 등 인증 제공자의 해시 저장을 사용하고 직접 평문 저장 금지
- 관리자 화면, 문제 검수 화면, 사용자 학습 기록에는 역할 기반 접근권한을 적용
- 접속 로그, 오류 로그, 관리자 변경 이력은 보관하되 목적과 기간을 정함
- 개인정보 유출 또는 오발송 상황에 대비해 신고/대응 절차를 문서화

## Phase 1 — 핵심 학습 기능 강화

> 수험생이 매일 쓰는 기능. 바로 체감 가능.

### 11. 과목 / 교시 / 문제 수 선택 퀴즈
- 원하는 과목(약리학, 내과학 등)을 골라서 퀴즈 시작
- 복수 선택 가능 (예: "약리학 + 병리학 30문제")
- 교시별 선택 가능 (예: "1교시 기초 과목만")
- 문제 수 선택 가능 (5 / 10 / 30문제 등)
- 현재 전 과목 랜덤만 가능 → 약점 집중 학습 불가

### 12. 시험일 D-day + 학습 플랜
- 수의사 국가고시 날짜 입력 → 남은 일수 표시
- 하루 권장 문제 수 자동 계산
- 남은 문제은행 기준 예상 회독 수 표시
- 대시보드 최상단에 고정 노출

### 13. 미니 모의고사 / 실전 모의고사 모드
- 1차: 과목별 또는 교시별 미니 모의고사
- 2차: 실제 시험과 동일한 문제 수 / 과목 배분
- 시간 제한 타이머
- 제출 후 과목별 점수 리포트
- 합격선 대비, 취약 과목 Top 5, 다시 풀 문제 추천
- 실전 감각 훈련에 필수

## Phase 2 — 학습 유지 및 품질

> 문제를 더 잘 기억하고, 문제은행 품질을 계속 높이는 기능.

### 14. 문제 신고 / 오류 제보
- 문제 오류, 해설 오류, 정답 오류 신고 버튼
- 관리자 검토 후 수정 반영
- 신고 많은 문제는 자동 비활성 후보로 표시
- 문제 품질 유지에 필수

### 15. 관리자 검수 화면
- 신고 문항, 자동 검증 실패 문항, 이미지 보류 문항을 한 화면에서 검토
- 문항 본문, 선택지, 정답, 해설, 과목, 출처 메타데이터, 검수 상태를 함께 표시
- 승인, 수정 필요, 비활성화, 재작성 요청을 빠르게 처리
- 운영 초반에는 완성형 CMS보다 내부 검수 큐에 집중

### 16. 중복 / 유사 문항 탐지
- 동일 회차 중복, 연도별 반복 출제, AI 재작성 과정에서 생긴 유사 문항을 탐지
- 임베딩 또는 텍스트 유사도 기준으로 중복 후보를 묶어 관리자 검수 대상으로 표시
- 완전 중복은 병합하거나 하나만 노출하고, 반복 출제는 `related_questions`로 연결

### 17. 문항 품질 지표
- 초기에는 AI/검수자 기준 난이도와 태그를 저장
- 실제 풀이 데이터가 쌓이면 `correct_rate`, `attempt_count`, `avg_time_spent`, `discrimination_index`를 계산
- 지나치게 쉽거나, 정답률이 비정상적으로 낮거나, 신고가 많은 문항은 재검수 후보로 자동 분류

### 18. 문제 북마크
- 정답이어도 다시 보고 싶은 문제 저장
- 오답 노트와 별개의 "즐겨찾기" 리스트
- 북마크 모아서 복습하는 세션

### 19. 학습 스트릭 강화
- 연속 학습 일수 배지 + 시각적 강조
- 오늘 목표 달성 여부 (N문제 완료)
- 현재 대시보드에 일부 있으나 존재감 약함

### 20. 수험생 팁 노출
- `community_notes`를 "수험생 팁" 섹션으로 노출
- 정정, 암기법, 보충 설명을 문제 하단에 표시
- 검수 전 댓글은 별도 표시 또는 관리자 승인 후 노출

## Phase 3 — 확장 기능

> 사용자 기반이 생긴 후 고려.

### 21. 운영 비용 / 사용량 모니터링
- Supabase, AI API, 이미지 생성, 스토리지, 이메일/푸시 비용을 월 단위로 추적
- 사용자 1명당 평균 문제 풀이 수, AI 호출 수, 스토리지 사용량을 기준으로 원가 구조 계산
- 무료 사용량, 유료 기능, 후원/구독 가능성을 비용 데이터 기반으로 판단
- 비용이 급증하는 기능은 캐싱, batch 처리, 관리자 승인 후 실행으로 제한

### 22. 업데이트 캘린더
- 매년 국가시험 시행 후 최신 회차 intake, 재작성, 검수, 반영 일정을 운영 루틴으로 정의
- 시험 직후에는 출제 경향 요약과 신규 회차 준비 상태를 사용자에게 투명하게 표시
- 최신 회차 반영 전에도 기존 문제은행의 품질 개선과 오답/신고 처리를 우선순위에 포함

### 23. 데이터 백업 / 장애 대응
- Supabase DB, 스토리지, import 원본 처리 산출물의 백업 주기 확정
- 운영 DB와 실험용 DB를 분리하고, 대량 변경은 staging에서 먼저 검증
- 사용자 풀이 기록, 오답노트, SRS 상태는 문제은행보다 더 강한 백업 기준 적용

### 24. 사용자 신뢰 / 투명성
- AI 재작성 문제은행임을 숨기지 않고, 검수 기준과 오류 제보 절차를 명확히 제공
- 정답 이의제기와 수정 이력을 사용자에게 납득 가능한 수준으로 공개
- "기출 원문 제공"이 아니라 "기출 기반 재구성 학습"이라는 포지셔닝을 일관되게 유지

### 25. 복습 알림
- 오늘 복습할 문제가 있을 때 이메일 알림
- 브라우저 Push 알림 (선택적)

### 26. 친구 / 학습 비교
- 친구 초대 링크
- 주간 문제 수 / 정답률 비교
- 학습 동기 부여용

### 27. AI 해설 강화
- 기존 해설 외에 AI가 추가 설명 생성
- "왜 틀렸는지" 개인화 피드백
- 선지별 오답 이유, 관련 개념, 비슷한 함정 설명
- 검수된 기본 해설을 우선하고 AI 해설은 보조 설명으로 제공
