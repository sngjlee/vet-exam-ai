## 포지셔닝 원칙

KVLE는 두 개의 레이어로 운영된다.

1. **공식 레이어**: 수의미래연구소 편집진이 책임지는 문제·해설·정정
2. **커뮤니티 레이어**: 수험생이 기여하는 암기법·토론·정정 제안·추가 설명

수의미래연구소는 **공식 콘텐츠 제공자**인 동시에 **커뮤니티 운영 주체**다. 두 역할은 UI·약관·DB에서 명확히 구분한다.

## Phase 0 — 문제은행 신뢰도 확보

> **진행 중.** Claude Code로 vet40 기출 → Sonnet 4.6 재작성 → 검수 → Supabase 삽입 파이프라인 운영.

### 1. 콘텐츠 출처 / 처리 이력 기록

- 각 문항의 원천, 재작성 모델, 프롬프트 버전, 처리 일시, 검수 상태를 추적
- `source_type`, `source_ref`, `rewrite_model`, `rewrite_prompt_version`, `review_status`, `reviewed_at` 같은 메타데이터
- 공개 서비스 전 문제 발생 시 문항 단위로 원인 추적·비활성화·재검수 가능해야 함
- 원문 텍스트·원본 이미지를 서비스 DB에 불필요하게 보관하지 않는 원칙

### 2. 최신 회차 커버리지 정책

- 67~70회는 최신 출제 경향 파악 위해 intake 대상
- 최신 회차 원문/이미지를 서비스 DB에 그대로 저장·노출하지 않음 (처리 이력과 재작성 후보만 관리)
- 회차별 수집 상태 추적: `not_started`, `intake_done`, `rewrite_done`, `reviewing`, `approved`, `blocked`
- 매년 시험 후 최신 회차 반영 루틴으로 반복 가능해야 함

### 3. AI 재작성 결과물 검수

- 과목별 샘플 5~10문제 원문 대비 확인
- 정답 유지, 선택지 중복/모순, 해설 정확도, 난이도 변형 점검
- 이미지 포함 문제는 1차에서 `is_active=false`로 분리

### 4. 이미지 포함 문항 권리 리스크 처리

- vet40 등 기출 아카이브 원본 이미지는 그대로 사용하지 않음
- 이미지 유형 태깅: `diagram_rebuild`, `licensed_asset_needed`, `question_rewrite_needed`, `remove_image`
- 해부도/생리도/표/알고리즘은 원본 구도에 의존하지 않는 새 도식으로 재작성
- 병변 사진, 방사선, 초음파, 조직 슬라이드는 라이선스 확보·공개 자료·자체 제작 또는 이미지 비의존 재설계
- 원본 이미지를 입력으로 한 단순 스타일 변환은 실질적 유사성 리스크가 남으므로 지양

### 5. 검수 상태 워크플로우

- 상태 분리: `raw_imported`, `ai_rewritten`, `auto_validated`, `human_reviewed`, `image_pending`, `approved`, `rejected`
- 노출 가능 상태와 내부 보류 상태를 명확히 구분
- 이미지 보류·정답 의심·해설 의심·과목 분류 의심을 별도 플래그로 관리
- 관리자가 상태 변경한 이력 보관(감사 로그)

### 6. 자동 검증 스크립트

- `choices.length === 5`
- `answer`가 choices 중 하나와 정확히 일치
- 문제/선택지/정답/해설 빈 값 검사
- `id`, `subject`, `category`, `session`, `round`, `year` 누락/중복 검사
- `review_status`, `source_type`, `rewrite_model`, `import_batch_id`, `exam_round` 등 필수 운영 메타데이터 검사
- 전체 JSON 검증 통과분만 업로드 후보

### 7. Supabase 스키마 확장 (문제 영역)

- `questions` 테이블에 `session`, `round`, `year`, `tags`, `is_active` 추가
- `review_status`, `source_type`, `source_ref`, `rewrite_model`, `rewrite_prompt_version`, `import_batch_id`, `asset_status`, `exam_round` 추가
- `source="past_exam"` 기준으로 기출 기반 문제 구분
- `id` 기준 upsert 가능하도록 중복 정책 확정

### 8. 대량 업로드 / 롤백 전략

- 모든 import를 `import_batch_id` 단위로 기록
- 업로드 전 dry-run으로 신규/수정/충돌/비활성화 예정 출력
- 배치 단위 비활성화·롤백 스크립트 준비
- 과목별/회차별 분할 업로드, 샘플 검수 통과 후 다음 배치

### 9. 20과목 데이터 구조 반영

- 기존 5과목 중심에서 수국시 20과목으로 확장
- 과목 선택, 통계, 약점 분석, 색상 팔레트가 20과목 처리
- 교시별 필터(1/2/3교시/법규) 확장 고려

### 10. 소량 업로드 후 앱 플로우 테스트

- 예: `해부학` + `수의법규` 2과목 먼저 업로드
- 퀴즈, 오답노트, SRS, 통계, 약점 집중 연습 정상 동작 확인

## Phase 0.5 — 공개 전 정책 및 UGC 기반

> 공개 서비스 전환 전 반드시 확정. **UGC 플랫폼 특성상 v1보다 대폭 확장됨.**

### A. 개인정보 최소수집 회원가입

- 초기 가입 필수 항목: 이메일, 비밀번호, 약관 동의, 개인정보처리방침 동의, **닉네임**
- 선택 항목: 준비 회차, 소속 대학, 자기소개 (프라이버시 설정으로 공개 토글)
- 이름, 휴대전화, 학번, 재학증명서, 수의대생 인증 자료는 기본 가입 단계에서 수집하지 않음
- 마케팅 수신, 맞춤형 분석, 연구/통계 활용은 선택 동의로 분리
- 탈퇴 시 계정 식별정보·학습 기록·**작성 댓글** 삭제/익명화 기준 사전 정의
  - 댓글은 기본 "작성자 탈퇴로 익명화" (내용 유지, 닉네임 `탈퇴한 사용자`로 변경)
  - 선택: 작성 댓글 전체 삭제 요청 가능

### B. UGC(댓글) 정책 (신규)

- **소유권**: 작성자에게 귀속
- **플랫폼 사용권**: 비독점·전세계·양도 가능·무상. 표시·보관·복제·인용(검색 결과 스니펫 등) 목적
- **작성자 보증**: 자신의 댓글이 타인의 저작권·인격권을 침해하지 않음을 보증
- **이의제기**: 권리자 신고 시 신속 삭제 + 작성자에게 반박권 부여
- **커뮤니티 가이드라인**: 별도 문서로 관리, 주요 금지 행위 명시
  - 개인정보 노출 (본인·타인), 특정 개인 지목 비방
  - 광고·홍보·상업적 모집
  - 저작권 침해 (교재/강의 통째로 복붙 등)
  - 국가시험 부정행위 조장 (단순 암기법은 OK, 실제 시험 중 교환 방식 논의 NG)
  - 욕설·혐오·차별

### C. 가입 대상 / 자격 인증 정책

- 초기 버전은 일반 가입 허용, 서비스 포지셔닝은 "수국시 대비"로 명확히 유지
- 수의대생 전용 기능이 필요해질 때만 별도 인증 배지 도입
- **합격생 인증은 V2에서 도입** (MVP 미포함)
  - 방식 우선순위: 면허번호 조회 API → 합격증 수동 검증 → 대학 이메일
  - 검증 후 증빙 이미지는 즉시 파기, 검증 플래그만 DB에 남김
- 장기적 확장 위해 `user_role`, `exam_track` 분리 설계

### D. 미성년자 / 아동 개인정보

- 만 14세 미만 가입 불가 (명시)
- 수국시 수험생 특성상 대부분 해당 없음
- 동물보건사 확장 시 재검토

### E. 보안 / 접근통제 기본선

- 비밀번호는 Supabase Auth 해시 저장, 평문 저장 금지
- 관리자·검수·학습 기록에 역할 기반 접근권한
- 접속·오류·관리자 변경 이력 보관, 목적/기간 명시
- 개인정보 유출 대응 절차 문서화
- **댓글 관련 추가 보안**:
  - 이미지 업로드 시 EXIF 제거, 위치 정보 자동 스크럽
  - 외부 링크 `rel="nofollow"` 기본, whitelist 외 경고
  - 개인정보 자동 필터 (전화번호/이메일/카톡ID 패턴)

### F. 정보통신망법 / 저작권법 대응 (신규)

- **임시조치 절차**: 명예훼손 신고 접수 → 24시간 내 블라인드 → 30일 후 결정 → 이의제기 창구
- **저작권 침해 대응**: 권리자 확인 신고 → 신속 삭제 → 작성자 반박권
- 약관에 의무 조항 명시, 처리 절차를 `docs/moderation-playbook.md`에 기재

### G. 관리자 페이지 MVP

- `/admin` 별도 페이지, 초기에는 완성형 CMS보다 운영 필수 도구로 제한
- 관리자 권한은 `user_role` 또는 allowlist로 분리, Supabase RLS + 서버 권한 검사
- **1차 범위**:
  - 가입자/권한 확인
  - 문제 업로드 배치 상태
  - 문항 검수 상태 변경
  - **댓글 신고 큐**
  - **정정 제안 큐**
  - 문제 비활성화
  - 뱃지 수동 부여 (검수자/운영자)
- 운영 설정: 다음 시험일 기본값, 공지/점검 메시지, 최신 회차 반영 상태
- 모든 관리자 변경은 `admin_audit_logs`에 기록

## Phase 1 — Solo 학습 MVP

> 수험생이 매일 쓰는 개인 학습 기능. 5월 중순까지 안정화.

### 11. 과목 / 교시 / 문제 수 선택 퀴즈

- 원하는 과목(약리학, 내과학 등) 선택 후 퀴즈 시작
- 복수 선택 가능 (예: "약리학 + 병리학 30문제")
- 교시별 선택 가능 (예: "1교시 기초 과목만")
- 문제 수 선택 가능 (5/10/30 등)
- 현재 전 과목 랜덤만 가능 → 약점 집중 학습 불가

### 12. 시험일 D-day + 학습 플랜

- 수국시 날짜 입력 → 남은 일수 표시
- 하루 권장 문제 수 자동 계산
- 남은 문제은행 기준 예상 회독 수
- 대시보드 최상단 고정 노출
- 관리자가 공식 시험일 입력하면 전 유저 기본값 갱신

### 13. 공식 해설 / 커뮤니티 탭 구조 (신규, F1 확장)

- 정답 확인 화면에 `공식 해설` / `커뮤니티 토론(N)` 탭
- 기본 선택 탭은 공식 해설 (초기 댓글 수 적을 때)
- 댓글 수가 임계치(예: 5개) 이상인 문제는 커뮤니티 탭에 약한 강조 표시
- 모바일에서는 탭 전환 스와이프

## Phase 2 — 커뮤니티 MVP (핵심 신규)

> **이 Phase가 KVLE를 vet40 시절의 집단지성과 연결하는 핵심이다.**
> 6월 말까지 완성하여 7월 1일 소프트 런칭에 맞춘다.

### 14. 댓글 코어 (F7)

- 문제별 댓글 스레드
- **에디터**: 텍스트 + 이미지 + 간단 서식(굵게/기울임/색상/리스트/인용)
  - 라이브러리: TipTap 유력 (V2에서 수식 확장 용이)
- **답글**: 1단계 (무한 뎁스 금지)
- **댓글 타입**: `memorization`, `correction`, `explanation`, `question`, `discussion`
- **이미지 첨부**: 파일당 5MB 이하, 댓글당 3장, 자동 압축 + WebP + EXIF 제거
- **수정 이력**: 수정됨 표시 + 변경 내역 열람
- **삭제**: 본인 삭제 시 "작성자에 의해 삭제됨" 표시

**수락 기준:**
- [ ] 댓글 작성·수정·삭제 RLS 통과
- [ ] 이미지 업로드·압축·표시 정상
- [ ] 답글 1단계 제한 동작
- [ ] 타인 댓글 수정/삭제 불가

### 15. 추천 / 정렬 / 신고 (F8)

- **추천/비추천**: 유저당 1회, 변경 가능
- **정렬**: 추천순(기본) / 최신순 / 오래된순
- **신고**:
  - 사유: 스팸, 오답 전파, 개인정보 노출, 욕설/혐오, 광고, 저작권 침해, 기타
  - 부가 설명 입력(선택)
- **자동 블라인드**: 신고 3회 시 즉시 블라인드 + 관리자 큐
- **비추천 임계치 접힘**: 순 비추천 -5 도달 시 접힘 상태, 클릭 시 열람

### 16. 유저 프로필 (F9)

## 2026-04-27 사용자 피드백 구현 로드맵

> 목표: 실제 사용자가 주로 쓰는 "해설보기 중심 학습"을 1순위로 개선한다. 댓글 고정, 개념별 탐색, 오답/쉬운문제 필터 공통화, 문제 번호 일관성, 모바일/iPad UX를 묶어서 진행한다.

### 우선순위

| 우선순위 | 작업 | 이유 | 권장 단계 |
|---|---|---|---|
| P0 | 문제 번호 표시 통일 | 전체보기/해설보기 번호 불일치로 학습 혼란 발생 | 즉시 |
| P0 | 해설보기 필터 확장 | 사용자가 랜덤/모의고사보다 해설보기로 주로 공부함 | 즉시 |
| P1 | 댓글 암기팁 고정 | 2회독/3회독 때 핵심 암기팁을 다시 찾는 비용 감소 | 다음 |
| P1 | 모바일/iPad 문제 탐색 개선 | 번호 버튼 오탭, 다음 문제 버튼 부재, 가독성 불편 | 다음 |
| P1 | 개념별 탐색 | 회차별 학습만으로는 1회독 진입장벽이 큼 | 다음 |
| P2 | 오류 메시지/세션 실패 개선 | 보안/인증/네트워크 오류 체감 개선 | 병행 |
| P2 | 빈출 개념 요약집 | 콘텐츠 검수 필요성이 크므로 별도 에픽 | 후속 |

### 구현 순서

1. **문제 표시 모델 정리**
   - `Question` 타입과 문제 UI에서 공식 문제 식별자와 세션 순번을 분리한다.
   - `QuestionCard`의 `Q001`은 공식 번호로 쓰지 않고, `현재 3 / 20` 진행 표시에만 사용한다.
   - `QuestionReadOnly`와 상세 페이지도 같은 표시 규칙을 따른다.

2. **공용 문제 필터 유틸 추가**
   - `오답문제만`, `쉬운문제생략`, `category`, `topic`, `round`, `session` 필터를 한 유틸로 모은다.
   - 랜덤문제, 해설보기, 전체보기, 추후 모의고사에서 같은 필터 결과를 사용한다.

3. **해설보기 목록/탐색 화면 강화**
   - 해설보기에서 전체 문제를 목록으로 볼 수 있게 하고 필터 바를 붙인다.
   - 필터 상태에서 이전/다음 문제 이동이 가능해야 한다.
   - 오답만/쉬운문제생략 버튼은 랜덤문제 전용이 아니라 해설보기에도 노출한다.

4. **댓글 고정 기능**
   - `comment_pins` 테이블을 추가한다.
   - 사용자별/문제별 고정 댓글을 저장한다.
   - MVP는 문제별 1개 고정으로 시작한다.
   - 댓글 목록 상단에 "내가 고정한 암기팁" 영역을 표시한다.

5. **모바일/iPad UX 개선**
   - 문제 번호/이동 버튼 터치 영역을 최소 44px로 맞춘다.
   - 해설보기와 풀이 화면에 `이전 문제`, `다음 문제` 버튼을 제공한다.
   - 텍스트 복사가 필요한 영역은 PC/iPad/모바일에서 선택 가능하게 한다.

6. **오류 처리 개선**
   - 인증 만료, RLS 권한 오류, 네트워크 오류, 서버 오류를 구분한다.
   - 문제/댓글 로딩 실패에는 재시도 버튼을 제공한다.
   - 콘솔 로그만 남기지 않고 사용자에게 이해 가능한 안내를 보여준다.

7. **빈출 개념 요약집 기획/데이터 준비**
   - `topic`, `tags`, `round`, 사용자 오답률, 검수된 암기팁을 후보 데이터로 모은다.
   - 공식 요약집은 운영자 검수 후 공개한다.
