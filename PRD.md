# PRD: 수의사 국가시험 모의고사 플랫폼

**버전:** 0.1 (초안)
**작성일:** 2026-03-17
**백엔드 담당:** Claude Code
**프론트엔드 담당:** Antigravity

---

## 1. 제품 개요

### 1.1 배경 및 목적

수의사 국가시험(이하 "수국시") 수험생이 충분한 양질의 문제로 반복 학습할 수 있는 환경이 부족하다. 기존 시중 교재나 스터디 자료는 문제 풀이 이후 복습 관리가 체계적이지 않으며, 개인별 약점 영역을 파악하고 집중 공략하는 기능이 없다.

본 제품은 수국시 수험생을 위한 **과학적 반복 학습(Spaced Repetition) 기반 모의고사 플랫폼**으로, 단순 문제 풀이를 넘어 오답 관리 → 복습 큐 → 적응형 학습의 사이클을 제공한다.

### 1.2 핵심 가치 제안

- **스마트 복습**: 틀린 문제를 기억 과학에 기반한 간격(1→3→7→14일)으로 자동 재출제
- **약점 집중**: 카테고리별 정답률 분석으로 가장 취약한 과목을 자동 파악, 집중 훈련
- **이어서 학습**: 비회원 → 회원 전환 시 기존 오답노트 자동 인계
- **실전 감각**: 수국시 출제 경향을 반영한 문제 구성 및 영역별 분류

## 3. 기능 요구사항

### 3.1 핵심 기능 (MVP)

#### F1. 문제 풀이 세션 (Quiz Session)

- 사용자가 카테고리(과목) 또는 "전체"를 선택 후 세션 시작
- 세션당 5문제 (추후 설정 가능하도록 확장 예정)
- 각 문제는 5지선다형 (수국시 실제 형식 반영)
- 답안 선택 → "정답 확인" → 정오 및 해설 즉시 표시
- 세션 종료 시 점수 및 오답 요약 표시
- 오답은 자동으로 오답노트에 저장

**DB 연관:** `questions`, `attempts` 테이블
**담당:** 백엔드(문제 API), 프론트엔드(퀴즈 UI)

---

#### F2. 오답노트 (Wrong Notes)

- 오답으로 저장된 문제 목록 열람
- 카테고리별 필터링
- 각 문제의 정답, 오선택지, 해설 표시
- 개별 항목 삭제 / 전체 삭제
- "선택 문제 다시 풀기" 기능 (오답노트 → 재시도 세션)
- 재시도 세션에서 정답 시 → 오답노트에서 제거

**DB 연관:** `wrong_notes` 테이블
**스토리지:** 비회원 = localStorage, 회원 = Supabase
**담당:** 백엔드(CRUD API + 스토리지 분기), 프론트엔드(오답노트 UI)

---

#### F3. 간격반복 복습 큐 (Spaced-Repetition Review)

- 복습 예정 항목 수를 NavBar에 배지(badge)로 표시
- 복습 큐: `next_review_at <= now()` 인 항목을 최대 5개씩 출제
- 복습 결과에 따라 다음 복습 일정 재계산:
  - 정답 → 다음 인터벌로 이동 (1 → 3 → 7 → 14일)
  - 오답 → 즉시 복습으로 리셋 (review_count = 0)
- 복습 완료 후 남은 큐 재조회

**DB 연관:** `wrong_notes` 테이블 (`review_count`, `last_reviewed_at`, `next_review_at`)
**담당:** 백엔드(스케줄 계산 로직 + API), 프론트엔드(복습 화면)

---

#### F4. 통계 대시보드 (My Stats)

- 전체 통계: 총 풀이 수, 정답 수, 정답률, 최근 7일 풀이 수
- 카테고리별 통계 테이블: 풀이 수 / 정답 수 / 정답률
- 최근 풀이 이력 (최대 20건)
- "약점 집중 연습" 버튼 → 적응형 연습으로 이동
- **로그인 필수**

**DB 연관:** `attempts` 테이블 집계
**담당:** 백엔드(통계 집계 API), 프론트엔드(차트 및 테이블 UI)

---

#### F5. 적응형 연습 (Adaptive Practice)

- 카테고리별 정답률 기반 최약 과목 자동 선정
  - 최소 3회 이상 시도한 카테고리 대상
  - 동점 시 오답 수 많은 쪽 우선
- 해당 카테고리 5문제 집중 출제
- **로그인 필수**

**DB 연관:** `attempts` 테이블
**담당:** 백엔드(약점 카테고리 분석 API), 프론트엔드(세션 UI)

---

#### F6. 인증 및 계정 관리 (Auth)

- 이메일/패스워드 회원가입 및 로그인 (Supabase Auth)
- 회원가입 시 프로필 자동 생성 (DB 트리거)
- 비회원 오답노트 → 로그인 시 자동 마이그레이션
- 로그아웃

**담당:** 백엔드(Supabase Auth 설정, 마이그레이션 API), 프론트엔드(로그인 UI)

## 4. 비기능 요구사항

### 4.1 성능
- 문제 로딩: 첫 렌더 후 200ms 이내 표시 (캐싱)
- 정답 제출 → DB 저장: 2초 이내
- 복습 큐 조회: 1초 이내

### 4.2 보안
- Row-Level Security(RLS): 사용자는 자신의 데이터만 접근 가능
- 문제 데이터: 인증된 사용자에게만 공개
- attempts 테이블: 삽입 전용 (수정/삭제 불가, 감사 로그)

### 4.3 접근성 및 UX
- 모바일 퍼스트 반응형 레이아웃
- 다크 테마 지원 (현재 기본값)
- 로딩 상태 및 에러 상태 명시적 표시

### 4.4 확장성
- 문제 은행: 초기 수십 문제 → 수천 문제 확장 고려
- 다중 시험 지원 구조 (향후 타 국가시험 확장 가능성)

## 6. 데이터 모델 요약

```
profiles          → 사용자 프로필 (auth.users 1:1)
questions         → 문제 은행 (category, subject, topic, difficulty, source)
attempts          → 풀이 이력 (세션 ID, 문제 ID, 정오, 시각)
wrong_notes       → 오답노트 + 복습 스케줄 (review_count, next_review_at)
```

**핵심 설계 원칙:**
- `wrong_notes` 비정규화: 문제 텍스트/선택지/해설 복사 저장 → 조인 없이 복습 가능
- `attempts` 불변(immutable): 삽입만 허용, 통계 기반 데이터
- RLS로 멀티유저 데이터 격리

## 7. 사용자 흐름 (User Flows)

### 핵심 플로우 A: 첫 방문 비회원
```
홈 방문 → 카테고리 선택 → 문제 풀기 (5문제)
→ 오답 자동 저장 (localStorage) → 오답노트 확인
→ 회원가입 → 오답노트 자동 마이그레이션 → 복습 큐 시작
```

### 핵심 플로우 B: 기존 회원 일일 학습
```
로그인 → 홈 화면에서 "복습 N개 예정" 확인
→ 복습 세션 (최대 5문제) → 완료
→ 새 문제 연습 or 약점 집중 연습 → 오늘 학습 종료
```

### 핵심 플로우 C: 통계 기반 전략 학습
```
My Stats 접속 → 카테고리별 정답률 확인
→ "약점 집중 연습" 클릭 → 최약 카테고리 5문제 세션
→ 오답 자동 복습 큐 등록 → 14일 후 완전 학습 완료
```
