## 1. 제품 개요

### 1.1 배경 및 목적

수의사 국가시험(이하 "수국시") 수험생은 두 가지 구조적 어려움에 부딪힌다.

1. **체계적 반복 학습 부재**: 기존 교재·스터디 자료는 풀이 후 복습 관리가 비체계적이며, 개인 약점 영역을 데이터로 파악할 수 없다.
2. **해설의 획일성**: 기출 해설은 표준 설명 한 가지에 그치고, 수험생들이 실제로 쓰는 암기법·오류 지적·대안적 접근은 개인 노트나 소규모 스터디에 흩어져 있다.

KVLE는 두 문제를 동시에 해결한다.

- **공식 레이어**: AI 재작성을 거친 검수된 기출 기반 문제은행 + SRS(SM-2) + 약점 분석 + 자동 오답노트
- **커뮤니티 레이어**: 문제별 암기법·정정·추가 해설·토론이 축적되는 수험생 기여 공간

### 1.2 핵심 가치 제안

| 가치 | 설명 | 레이어 |
|---|---|---|
| **스마트 복습** | 틀린 문제를 SM-2 기반 간격(1→3→7→14일)으로 자동 재출제 | 공식 |
| **약점 집중** | 카테고리별 정답률 분석으로 최취약 과목 자동 선별 | 공식 |
| **살아있는 해설** | 공식 해설 + 수험생 토론·암기법·정정으로 한 문제를 다각도로 이해 | 커뮤니티 |
| **집단 검증** | 오답·오해설은 다른 수험생의 지적과 추천으로 빠르게 바로잡힘 | 커뮤니티 |
| **이어서 학습** | 비회원 → 회원 전환 시 기존 학습 기록 자동 인계 | 공식 |

### 1.3 포지셔닝 원칙

- 랜딩/브랜딩에서 **"공식 문제은행 + 수험생 커뮤니티"** 하이브리드 메시지를 일관 유지
- "검증된 기출 기반 + 수험생이 실제로 외우는 방식을 아는 곳"
- 공식 콘텐츠의 신뢰도와 커뮤니티의 풍부함을 동시에 전달

## 2. 목표 사용자

### 2.1 세그먼트

| 세그먼트 | 설명 | 비중 목표 |
|---|---|---|
| **핵심 소비자** | 수의사 국가시험 준비 중인 수의대 본4~본6 | ≥70% |
| **부가 소비자** | 재응시 준비생, 지식 유지 원하는 현직 수의사 | ≥15% |
| **적극 기여자** | 상위 학년, 합격생, 과목별 스터디 리더 | ≥10% (품질 결정) |
| **비회원 열람자** | 검색 유입, 가입 전환 타깃 | 가능한 많이 |

### 2.2 페르소나

**"수국시 준비생 김○○" (본4, 핵심 소비자)**
- 하루 1~2시간 모바일/PC 양쪽에서 문제 풀이
- 번식학/내과학 취약점 인지
- 기존 해설만으로 이해 안 되는 문제에서 막힘
- 다른 수험생 암기법·정리 참고하고 싶어함
- 자기도 좋은 팁 있으면 공유 의향 있음

**"고학년 기여자 이○○" (본6, 적극 기여자)**
- 본인은 이미 회독 완료, 약한 과목만 재확인
- 후배 도우면서 본인도 복습 효과
- 양질의 댓글 반복 작성 → 뱃지·평판 획득 동기
- 다음 해 합격 후 "합격생 뱃지" 기대

**"재응시 준비생 박○○"**
- 작년 불합격, 취약 과목 집중 공략 필요
- 과거 스터디 경험으로 암기법 여러 개 보유
- 기여 의욕과 소비 의욕 모두 높음

## 3. 수의미래연구소의 역할

**핵심 원칙: 하나의 조직이 두 가지 역할을 수행하되, UI·ToS·DB에서 레이어를 명확히 분리한다.**

### 3.1 공식 콘텐츠 제공자 (편집진)

- 문제, 1차 해설, 과목 분류, 정답에 대한 최종 결정권
- 문제·해설 저작권 주체 (현 footer 문구 유지)
- AI 재작성 결과물 검수 주체
- 공식 정정 반영 결정권

### 3.2 커뮤니티 운영 주체 (운영팀)

- 댓글 가이드라인 수립 및 집행
- 신고 처리, 모더레이션
- 유저 뱃지 부여
- 정정 워크플로우 최종 판정
- UGC 소유권은 작성자에게 귀속, 플랫폼은 비독점적 사용권만 보유

### 3.3 UI 계층 구분

| 계층 | 표시 방법 | 색상/배지 |
|---|---|---|
| 공식 해설 | "수의미래연구소 편집진" 명의 | 은색 배지, 상단 고정 |
| 운영진 댓글 | "운영자" 배지 | 일반 댓글 영역, 눈에 띄게 |
| 검수자 댓글 | "검수자" 배지 | 일반 댓글 |
| 뱃지 보유 유저 댓글 | 해당 뱃지 | 일반 댓글 |
| 일반 유저 댓글 | 닉네임만 | 일반 댓글 |

## 4. 기능 요구사항

### 4.1 MVP (여름 런칭 범위)

#### F1. 문제 풀이 세션 (Quiz Session)

- 카테고리(과목) 또는 "전체" 선택 후 세션 시작
- 세션당 5문제 (추후 설정 가능)
- 5지선다형 (수국시 실제 형식)
- 답안 선택 → "정답 확인" → 정오 + 공식 해설 즉시 표시
- **정답 확인 화면에 "공식 해설" / "커뮤니티 토론(N)" 탭 노출**
- 세션 종료 시 점수 + 오답 요약
- 오답 자동 오답노트 저장

**DB:** `questions`, `attempts`
**담당:** 백엔드(문제 API), 프론트엔드(퀴즈 UI + 탭 구조)

#### F2. 오답노트 (Wrong Notes)

- 오답 목록 열람, 카테고리별 필터
- 정답·오선택·공식 해설 표시
- 개별/전체 삭제
- "선택 문제 다시 풀기" (재시도 세션)
- 재시도에서 정답 시 오답노트에서 제거

**DB:** `wrong_notes`
**스토리지:** 비회원 = localStorage, 회원 = Supabase
**담당:** 백엔드(CRUD + 스토리지 분기), 프론트엔드(UI)

#### F3. 간격반복 복습 큐 (SRS)

- NavBar에 복습 예정 수 배지
- `next_review_at <= now()` 항목을 최대 5개씩 출제
- 결과에 따라 다음 일정 재계산 (정답: 1→3→7→14일 / 오답: 즉시 리셋)

**DB:** `wrong_notes` (`review_count`, `last_reviewed_at`, `next_review_at`)
**담당:** 백엔드(스케줄 로직 + API), 프론트엔드(복습 화면)

#### F4. 통계 대시보드 (My Stats)

- 전체 통계: 총 풀이 수, 정답 수, 정답률, 최근 7일 풀이 수
- 카테고리별 통계 테이블
- 최근 풀이 이력 (최대 20건)
- **기여 통계**: 내가 쓴 댓글 수, 받은 추천 수, 획득 뱃지
- "약점 집중 연습" 버튼 → 적응형 연습
- 로그인 필수

**DB:** `attempts`, `comments`, `comment_votes`, `badges` 집계
**담당:** 백엔드(집계 API), 프론트엔드(차트/테이블)

#### F5. 적응형 연습 (Adaptive Practice)

- 최소 3회 이상 시도한 카테고리 대상, 정답률 기준 최약 선정
- 동점 시 오답 수 많은 쪽 우선
- 해당 카테고리 5문제 집중 출제
- 로그인 필수

**DB:** `attempts`
**담당:** 백엔드(분석 API), 프론트엔드(세션 UI)

#### F6. 인증 및 계정 관리 (Auth)

- 이메일/패스워드 회원가입·로그인 (Supabase Auth)
- 가입 시 프로필 자동 생성 (DB 트리거)
- 비회원 오답노트 → 로그인 시 자동 마이그레이션
- 로그아웃
- **가입 필수 항목 최소화**: 이메일·비밀번호·약관 동의·개인정보처리방침 동의·닉네임
- **선택 항목**: 준비 회차, 소속 대학(비공개 설정 가능), 자기소개
- 소셜 로그인은 V2 이후

**담당:** 백엔드(Auth 설정, 마이그레이션 API), 프론트엔드(로그인/가입 UI)

---

#### F7. 댓글/토론 시스템 (신규 MVP)

문제별로 커뮤니티 토론 스레드를 가진다.

- **작성 대상**: 모든 공개 문제
- **에디터**: 텍스트 + 이미지 첨부 + 간단 서식(굵게/기울임/색상/리스트/인용)
  - 수식(KaTeX)은 V2
  - 표는 V2
- **답글**: 1단계만 허용 (무한 뎁스 금지)
- **댓글 타입**: `memorization`, `correction`, `explanation`, `question`, `discussion` 중 작성 시 선택
- **이미지 첨부**:
  - 파일당 최대 5MB, 댓글당 최대 3장
  - 업로드 시 자동 압축(2000px 이하) + WebP 변환
  - Supabase Storage 사용
- **댓글 수정 이력**: 수정됨 표시 + 변경 내역 열람 가능
- **삭제**: 본인 삭제 시 "작성자에 의해 삭제됨" 표시 (답글 있으면 내용만 숨김)
- **인용 답글**: 원 댓글 특정 문구 인용 후 답글 작성 가능

**DB:** `comments`, `comment_edit_history`
**담당:** 백엔드(API + 이미지 업로드 + validation), 프론트엔드(에디터 + 스레드 UI)

**수락 기준(Acceptance):**
- [ ] 텍스트 댓글 작성·수정·삭제 동작
- [ ] 이미지 첨부 업로드·자동 압축·표시 동작
- [ ] 답글 1단계 정상 동작, 2단계 시도 시 불가
- [ ] 수정 이력 열람 가능
- [ ] 본인 외 타인 댓글 수정·삭제 불가 (RLS)

#### F8. 추천 / 정렬 / 신고

- **추천/비추천**: 댓글당 유저 1회, 변경 가능
- **정렬 옵션**: 추천순(기본) / 최신순 / 오래된순
- **댓글 신고**:
  - 사유: 스팸, 오답 전파, 개인정보 노출, 욕설/혐오, 광고, 저작권 침해, 기타
  - 사유 + 부가 설명(선택) 입력
- **임계치 자동 숨김**: 신고 3회 누적 시 자동 블라인드 + 관리자 큐
- **정보통신망법 임시조치**: 명예훼손 관련 신고는 30일 블라인드 절차 적용
- **비추천 임계치 숨김**: 순 비추천 수 -5 도달 시 "숨겨진 댓글" 접힘 상태, 클릭 시 열람 가능

**DB:** `comment_votes`, `comment_reports`
**담당:** 백엔드(투표/신고 API + 자동 숨김 로직), 프론트엔드(UI + 접힘 상태)

#### F9. 유저 프로필 (공개)

- 닉네임, 자기소개, 준비 회차, 가입일
- 내가 쓴 댓글 목록 (최근순), 받은 추천 합계
- 획득 뱃지 (MVP 범위):
  - **운영자** (수동 부여)
  - **검수자** (수동 부여)
  - **새내기** (가입 후 자동)
  - **첫 기여** (첫 댓글 작성 시)
  - **인기 댓글** (단일 댓글 추천 10회 이상)
- 프라이버시 설정: 준비 회차·대학 공개 여부 토글
- **합격생 뱃지는 V2에 도입** (MVP엔 없음)

**DB:** `user_profiles_public`, `badges`
**담당:** 백엔드(프로필 API + 뱃지 자동 부여 트리거), 프론트엔드(프로필 페이지)

#### F10. 알림 시스템 (최소)

- 인앱 알림 센터 (NavBar 벨 아이콘 + 배지)
- 알림 유형:
  - 내 댓글에 답글 달림
  - 내 댓글 추천 받음 (첫 추천 + 10/50/100개 마일스톤)
  - @멘션됨 (V2에서 활성화)
  - 내가 신고한 댓글 처리 결과
  - 내 댓글이 임시조치/블라인드 처리됨
- 읽음/안읽음 상태
- **이메일 알림은 V2** (인앱만 MVP)

**DB:** `notifications`
**담당:** 백엔드(알림 생성 트리거 + 조회 API), 프론트엔드(알림 센터)

## 5. 비기능 요구사항

### 5.1 성능

- 문제 로딩 첫 렌더: 200ms 이내 (캐싱)
- 정답 제출 → DB 저장: 2초 이내
- 복습 큐 조회: 1초 이내
- **댓글 스레드 로딩: 문제 페이지 진입 후 500ms 이내 (50개 이하)**
- **댓글 작성 → 표시: 1초 이내**

### 5.2 보안

- Supabase Row-Level Security (RLS)로 사용자 데이터 격리
- `attempts` 테이블 삽입 전용 (수정/삭제 불가)
- 비밀번호는 Supabase Auth 해시 저장 (평문 저장 금지)
- 관리자 화면·검수 화면·사용자 학습 기록은 역할 기반 접근 권한
- 접속/오류/관리자 변경 로그 보관 (기간·목적 명시)

### 5.3 UGC 및 법적 요구사항

- **이용약관 / 개인정보처리방침**: 공개 전 확정, 탈퇴·파기 기준 포함
- **UGC 조항**:
  - 작성자 소유권 보유
  - 플랫폼에 비독점·전세계·양도 가능한 사용권 부여 (표시·보관·모더레이션 목적)
  - 작성자는 자신의 댓글에 대해 원저작권 위반하지 않음을 보증
- **정보통신망법 임시조치 절차**: 명예훼손 신고 시 30일 블라인드 + 이의제기 창구
- **저작권 침해 댓글 대응**: 권리자 신고 → 신속 삭제 → 반박권 부여
- **개인정보 자동 필터**: 댓글 내 전화번호/이메일/카톡ID 패턴 감지 시 경고
- **외부 링크 정책**: whitelist 외 `rel="nofollow"` + 경고
- **미성년자**: 만 14세 미만 가입 불가 (명시). 준비 회차 특성상 거의 해당 없음

### 5.4 모더레이션 응답 시간

- 자동 블라인드(3회 신고): 즉시 (시스템)
- 명예훼손 임시조치: 접수 후 24시간 내
- 일반 신고 검토: 접수 후 72시간 내
- 정정 제안 검토: 추천 임계치 도달 후 1주 내

### 5.5 접근성 및 UX

- 모바일 퍼스트 반응형
- 다크 테마 기본 지원
- 로딩/에러 상태 명시적 표시
- WCAG 2.1 AA 수준 준수 노력
- 키보드 내비게이션

### 5.6 확장성

- 문제은행: 초기 수백 → 수천 문제 확장
- 댓글: 문제당 댓글 수 증가에도 페이지네이션으로 대응
- 다중 시험 지원 구조 (`exam_track` 필드로 향후 동물보건사 등)

## 7. 데이터 모델

```
profiles              사용자 프로필 (auth.users 1:1, private)
user_profiles_public  공개 프로필 (닉네임, 자기소개, 준비 회차 등)

questions             문제 은행 (category, subject, topic, difficulty, source, 재작성 메타)
attempts              풀이 이력 (immutable)
wrong_notes           오답노트 + SRS 스케줄

comments              댓글 (question_id, user_id, parent_id, body, type, status)
comment_votes         추천/비추천 (comment_id, user_id, value ±1)
comment_reports       신고 (comment_id, reporter_id, reason, status)
comment_edit_history  댓글 수정 이력

badges                유저 뱃지 (user_id, badge_type, awarded_at, reason)
notifications         알림 (user_id, type, payload, read_at)

question_corrections  정정 제안 (question_id, proposed_by, status, resolved_at)
admin_audit_logs      관리자 활동 이력 (admin_id, action, target, before, after, at)
```

**핵심 설계 원칙:**

- `wrong_notes` 비정규화: 문제 텍스트/선택지/해설 복사 저장 → 조인 없이 복습
- `attempts` 불변: 삽입 전용
- `comments`는 정규화 유지 (수정 빈도 높음)
- `user_profiles_public`은 `profiles`와 분리해 공개 데이터와 민감 데이터 격리
- RLS로 멀티유저 격리 + 관리자 역할 기반 접근
- `admin_audit_logs`로 대량 수정/권한 오남용 추적

**주요 상태 필드:**

- `questions.review_status`: `raw_imported | ai_rewritten | auto_validated | human_reviewed | image_pending | approved | rejected`
- `comments.status`: `visible | hidden_by_author | hidden_by_votes | blinded_by_report | removed_by_admin`
- `comment_reports.status`: `pending | reviewing | upheld | dismissed`
- `question_corrections.status`: `proposed | reviewing | accepted | rejected`

상세 스키마(컬럼/인덱스/RLS 정책/트리거)는 별도 문서 `docs/schema.sql`로 관리.

## 8. 사용자 흐름

### 플로우 A: 첫 방문 비회원

```
홈 → 카테고리 선택 → 5문제 풀이 → 오답 자동 저장(localStorage)
→ 오답노트 확인 → 회원가입 → 오답노트 자동 마이그레이션
→ 복습 큐 시작
```

### 플로우 B: 기존 회원 일일 학습

```
로그인 → 홈에서 "복습 N개 예정" 확인 → 복습 세션(최대 5)
→ 새 문제 연습 or 약점 집중 연습 → 종료
```

### 플로우 C: 통계 기반 전략 학습

```
My Stats → 카테고리별 정답률 확인 → "약점 집중 연습"
→ 최약 카테고리 5문제 → 오답 자동 복습 큐 등록
```

### 플로우 D: 공식 해설로 부족한 경우 (신규)

```
문제 풀이 → 정답 확인 → 공식 해설 이해 안 감
→ "커뮤니티 토론" 탭 → 추천 높은 암기법·설명 발견
→ 추천 누름 → 다음 회독 시 이 문제에 본인 추천 이력 표시
```

### 플로우 E: 오류 정정 (신규)

```
문제 풀이 → 해설 오류 발견 → 댓글 타입 "정정"으로 작성
→ 다른 수험생 추천 누적 → 임계치 도달 → 관리자 검토 큐
→ 관리자 승인 → 문제/해설 수정 + 정정 이력 공개 + 제보자 뱃지/알림
```

### 플로우 F: 신규 기여자 유입 (신규)

```
SEO 검색으로 문제 페이지 유입(비회원) → 댓글 읽음
→ "나도 좋은 암기법 있는데" → 가입 → 첫 댓글 작성
→ "첫 기여" 뱃지 획득 → 추천 받음 → "인기 댓글" 뱃지
```

### 플로우 G: 댓글 신고 (신규)

```
부적절 댓글 발견 → 신고 버튼 → 사유 선택 → 제출
→ 임계치 전이면 관리자 큐 대기, 임계치 넘으면 자동 블라인드
→ 관리자 검토 → 결과 알림 (신고자·작성자 양쪽)
```
